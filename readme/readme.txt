配置一些外设功能的时候一定要看参考手册，完整的看完所需要的功能那块的描述，因为比较分散需要自己总结



IIC协议可以用片外外设IIC控制器去驱动，但是由于时钟不准确，会导致驱动的问题，所以选择模拟IO口驱动


在配置结构体时，最好配完所有结构体成员默认值为默认值，而不要不写，它不一定是默认值，可以不写的会有特别说明
比如串口的库函数得配置无硬件流，否则打印不了东西
比如定时器的数字滤波器分频要配置不分频，否则IIC延时时序会出问题
以上的解决办法是配置的时候清零结构体，因为结构体不一定是默认状态的
但是定时器的结构体滤波器分频是不必须要配的 如果不灵敏可尝试配置一下

IIC的驱动函数最好不要用__NOP延时函数，因为他是内核时钟，频率是84M，而IIC的接口在总线APB1上，最好用同一个总线上的定时器延时


库函数的引脚复用要在初始化GPIO结构体之前配置，否则会在打印前多一个？

BS8116电容按键手册没给一个重要的信息，就是它的按键高低位寄存器，高位的和低位的寄存器的最高位都是1，即复位值是0X8080


语音模块在配置完IO后要给个延时在进行驱动，才可以正常工作


IIC的引脚是有区别拉高拉低的引脚不一定是同一个引脚，所以IIC不是通用的在宏定义中，然后每个模块看引脚单独写IIC时序
写IIC要注意引脚，引脚不同要单独写IIC文件



在配置结构体时要想省步骤或上几个值必须看下它的宏定义值，有时不一定就像IO配置一样某一位是1就可配置的
比如不可以|上EXTI1_IRQn，因为上EXTI1_IRQn是6，EXTI1_IRQn是7，或上的结果不是正确的结果，那么就会配置功能出现异常


烧写AT指令库时，得把跳线帽切换成wifi_tx---cp2_rx  wifi_rx---cp2_tx  让上位机直接发数据给ESP32
烧完之后按EN键复位可能会出现 waiting for download ，把串口调试工具的DTR勾选取消掉即可，然后再助手上显示ready表示烧写成功
